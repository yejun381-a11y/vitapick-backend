# VitaPick — AI 영양제 추천 쇼핑몰 (백엔드)

내 몸에 맞는 영양제를 AI 챗봇이 추천해주고, 바로 주문까지 할 수 있는 쇼핑몰입니다.
4명이 함께 만든 팀 프로젝트이고, 저는 **팀장**을 맡아 아래 기능들과 **서버 배포 전체**를 담당했습니다.

<br>

## 담당한 부분

| 구분 | 내용 |
|------|------|
| 기능 | 상품(목록·상세·인기·신상품·검색), 찜하기, 리뷰, AI 챗봇, 마이페이지 일부 |
| 인프라 | AWS 서버 배포 전체 (팀장) |

<br>

## 신경 쓴 부분

### 상품 이미지 테이블 분리
상품 하나에 이미지가 목록용·상세용으로 두 장씩 들어갑니다. 이미지를 상품 테이블에 같이 넣으면 구조가 지저분해져서, 이미지만 따로 테이블로 분리해 상품 테이블과 연결했습니다. 어떤 이미지가 목록용인지 상세용인지는 구분 값을 따로 둬서 처리했습니다.

### AI 챗봇에 영양 성분 정보 전달
챗봇이 영양 성분을 근거로 추천하도록, 상품 테이블에 영양 성분 정보를 담는 컬럼을 추가했습니다. AI에게 질문을 보낼 때 우리 상품 목록과 이 성분 정보를 함께 넘겨줘서, 실제로 파는 상품을 성분에 맞게 추천하게 만들었습니다. 추천 결과는 정해진 형식(JSON)으로 받아 실제 상품 페이지와 바로 연결되도록 했습니다.

### API 키 환경변수로 분리
OpenAI API 키를 코드에 직접 적으면 노출되니까, 환경변수로 빼서 관리했습니다. 코드에는 키가 들어가지 않아 깃허브에 올려도 안전합니다.

### BOM 방식으로 버전 관리
여러 라이브러리 버전을 하나하나 적지 않고, Spring Boot가 제공하는 BOM 방식을 활용해 버전을 한곳에서 자동으로 맞추도록 했습니다. 버전 충돌 걱정 없이 관리가 쉬워집니다.

### 리뷰는 구매자만 작성
아무나 리뷰를 못 쓰도록, 그 상품을 실제로 주문한 적이 있는 사람인지 주문 내역을 확인한 다음에만 리뷰를 쓸 수 있게 했습니다.

### JWT 토큰 기반 로그인
로그인하면 AccessToken(5분)과 RefreshToken(10일) 두 가지를 발급합니다. AccessToken으로 인증하고, 만료되면 RefreshToken으로 자동 재발급해 로그인 상태가 끊기지 않게 했습니다. 토큰을 짧게 두고 자동 갱신하는 방식으로 보안과 사용성을 모두 잡았습니다.
